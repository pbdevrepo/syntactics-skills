# Pre-Writing Intake

Before writing any page or article, confirm you have the following. If info is missing, ask the user
or note gaps with `[FILL IN]` in the deliverable. Never invent specific facts (founding year, project
count, certifications).

---

## Required for All Modes

- [ ] **Client/brand name** - exact company name as it should appear in content
- [ ] **Primary keyword** - one search term this piece targets
- [ ] **Target audience** - who is reading this? (e.g., "SME owners in the Philippines")
- [ ] **Brand voice** - formal, conversational, authoritative? Any words or phrases to avoid?

---

## Required for Static Pages (Mode 1)

- [ ] **Founding year** - needed for entity descriptions and E-E-A-T
- [ ] **Location** - city, country (for local SEO and entity context)
- [ ] **Core services** - specific list, not "various services"
- [ ] **Tech stack** - platforms, languages, or tools the company uses
- [ ] **Key proof points** - project count, client count, awards, certifications
- [ ] **Existing pages to link to** - 2-3 internal link targets

---

## Required for Articles (Mode 2)

- [ ] **Article type** - how-to, listicle, pillar, opinion, or news
- [ ] **Target word count** (if specified by client)
- [ ] **CTA destination** - which service page or contact page to link to at the end

---

## Required for Audits (Mode 3)

- [ ] **URL or page content** - the existing copy to audit
- [ ] **Page type** - homepage, service page, blog post, about page
- [ ] **Known issues** - anything the user already knows is broken or underperforming

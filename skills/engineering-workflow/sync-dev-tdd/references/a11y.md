# Accessibility Testing

## Two-layer model

| Layer | Tool | When |
|---|---|---|
| Component (unit) | `jest-axe` | After GREEN on any FE component |
| Page (E2E) | `axe-playwright` / `@axe-core/playwright` | After each Playwright interaction in Step 5 |

## Coverage ceiling

Automated tools catch approximately 57% of accessibility issues. Automated testing is a floor, not a ceiling — flag for manual testing alongside automated results.

## Component-level: jest-axe

```typescript
import { render } from '@testing-library/react';
import { axe, toHaveNoViolations } from 'jest-axe';

expect.extend(toHaveNoViolations);

test('MyComponent has no a11y violations', async () => {
  const { container } = render(<MyComponent />);
  const results = await axe(container);
  expect(results).toHaveNoViolations();
});
```

Run this after GREEN on any component that renders interactive or structural HTML. Treat a11y failures as RED — fix before writing the next test.

## Page-level: axe-playwright

```typescript
import { checkA11y, injectAxe } from 'axe-playwright';

test('page has no critical a11y violations', async ({ page }) => {
  await page.goto('/path');
  await injectAxe(page);

  // After each interaction that renders new content:
  await page.getByRole('button', { name: 'Open dialog' }).click();
  await checkA11y(page, undefined, {
    runOnly: { type: 'tag', values: ['wcag2a', 'wcag2aa'] },
  });
});
```

## Severity levels

| Severity | CI action |
|---|---|
| critical / serious | Fail the test — block merge |
| moderate | Warn — surface in output, do not block |
| minor | Warn — surface in output, do not block |

Only `critical` and `serious` violations are blocking gates. `moderate` and `minor` are surfaced as warnings for manual triage.

## What automated testing does NOT catch

- Logical reading order that looks correct to the parser but confuses screen reader users
- Focus trap issues in complex interactions
- Color contrast in dynamically themed UIs
- Cognitive accessibility (plain language, error message clarity)

Complement with manual testing using a screen reader (NVDA, VoiceOver) on critical user journeys.

## Engineering Skill Behavior

`sync-dev-tdd` reads this file during Step 0 setup. When MCPs with framework relevance are present
(framework:laravel, framework:shadcn, etc.), those conventions are referenced during Planning.
When local skills are present, they are surfaced to the developer during the planning phase.

`sync-qa-runner` reads this file during Step 1b framework detection. When a `testing:e2e` MCP
(Playwright or Cypress) is registered, the runner switches to **MCP mode** — tests are executed
via direct MCP tool calls instead of generating spec files and running the CLI.

# Question Bank — Requirement Analyzer

Pool of clarifying questions for Phase 3. Select only what the input did not answer.
Never ask all of these — aim for 5–10, max 15.

---

## Modules & Scope

- Q-MS1: Is [module] a separate screen/page or part of another module?
- Q-MS2: Should [module] be available on mobile, desktop, or both?
- Q-MS3: Are there any modules you know are out of scope for this phase?
- Q-MS4: Should [ambiguous module] support bulk operations (bulk upload, bulk approve, etc.)?
- Q-MS5: Is [inferred module] part of this project or a future phase?

## User Roles

- Q-UR1: Who are the primary users of the system — internal staff, external clients, or both?
- Q-UR2: Does the Admin role have full access to everything, or are there restrictions?
- Q-UR3: Will clients/customers have their own login, or is this internal-only?
- Q-UR4: Are there sub-roles within [role] (e.g., read-only vs. edit access)?
- Q-UR5: Who manages user accounts — an admin, or self-registration?

## Integrations & Third Parties

- Q-IN1: Does the system need to connect to any existing software the client already uses?
- Q-IN2: Which payment gateway should be used — Stripe, PayPal, PayMongo, or other?
- Q-IN3: Should email/SMS notifications be sent, and through which provider?
- Q-IN4: Is there an existing API the system must integrate with?
- Q-IN5: Should data sync with any accounting, CRM, or ERP system?

## Business Rules

- Q-BR1: What happens when [action] — is there an approval step before it takes effect?
- Q-BR2: Are there any automated rules (e.g., escalate if no response in 48 hours)?
- Q-BR3: What data must be required vs. optional in [form/module]?
- Q-BR4: Are there any compliance or regulatory requirements (data privacy, HIPAA, etc.)?
- Q-BR5: How should the system handle [edge case — e.g., duplicate entries, expired records]?

## Technical Constraints

- Q-TC1: Is there a preferred tech stack, or is that open?
- Q-TC2: Does this need to integrate with an existing codebase or system?
- Q-TC3: Are there performance expectations (e.g., must support X concurrent users)?
- Q-TC4: Is there a preferred hosting environment (cloud, on-premise, specific provider)?
- Q-TC5: Are there any browser or device requirements?

## Timeline & Budget

- Q-TL1: Is there a target launch date or a hard deadline?
- Q-TL2: Should this be delivered in phases, or all at once?
- Q-TL3: Is there a defined budget range for this project?

## Ambiguous or Inferred Modules

- Q-AM1: You mentioned [vague phrase] — can you describe what that means in more detail?
- Q-AM2: When you said [term], did you mean [interpretation A] or [interpretation B]?
- Q-AM3: Is [inferred module] something you expect in this version, or is it a future consideration?
